### **Unit I: Software Engineering and Project Management**

**Core Concept:**
Sir/Mam, Software Engineering vaneko code lekhnu matra haina. User lai k chaincha bujhera, systematic tarika le, budget ra time vitra, high-quality software banaune process ho.

---

### **1.1 Nature and Characteristics of Software**

Software Hardware jasto hudaina. Yesko unique nature huncha:
1.  **Software is developed, not manufactured:** Hardware factory ma banchan (Assembly line), Software dimag le banaucha (Logical process).
2.  **Software doesn't "wear out" (Khidaina):** Hardware purano vayesi ghisrincha/bigrincha. Software physical haina, tei vayera wear out hudaina. Tara, change garda garda complex vayera **"Deteriorate"** (bekaar) huna sakcha.
3.  **Custom Built:** Dherai jaso software client ko requirement anusar banaincha (Tailor-made).

---

### **1.2 Software vs System Engineering**

*   **System Engineering:** Yo "Baau" (Parent) ho. Yesle **Whole System** (Hardware + Software + People + Database + Process) lai hercha.
*   **Software Engineering:** Yo "Chhora" (Child) ho. Yesle System vitra ko **Software Component** lai matra focus garcha.

---

### **1.3 Software Crisis and Myths (Viva Favorite)**

**A. Software Crisis (Sankat):**
1960s tira software project haru fail huna thale. Kina?
*   Project **Over Budget** jane.
*   Time ma nasakine (**Late**).
*   Quality naramro hune (**Bugs**).
*   Maintain garna garo hune.
*   *Solution:* Yehi crisis solve garna "Software Engineering" subject aako ho.

**B. Software Myths (Bhram haru):**
Manchhe haru k galat sochchan?
1.  **Management Myth:** "Hami sanga book of standards chha, tei vaye pugcha." (*Reality:* Developer le tyo book padhdaina, process practical huna parcha).
2.  **Customer Myth:** "Requirement pachi change garda ni huncha." (*Reality:* Pachi change garda cost 10x badhcha).
3.  **Practitioner (Developer) Myth:** "Code lekhyo, kaam sakkyo." (*Reality:* Code lekhepachi testing ra maintenance ma 60% time lagcha).
4.  **The "Manchhe Thapne" Myth (Brooks' Law):** "Project late vayo vane manchhe thapdinus, chito huncha." (*Reality:* Late project ma manchhe thapyo vane jhan late huncha - communication overhead le garda).

---

### **1.4 Four Ps of Software Project Management**

Project manage garna 4 wota pillar chaincha (**P-P-P-P**):
1.  **People:** Sab vanda important. Developers, Managers, Stakeholders. Team ramro chaincha.
2.  **Product:** K banaune? Scope k ho? Objectives clear huna parcha.
3.  **Process:** Kun model use garne? (Agile, Waterfall, Spiral?).
4.  **Project:** Planning garne, budget milaune, risk herne.

---

### **1.5 & 1.6 Metrics, Measures, and Indicators**

Yiniharu ustai suninchha tara farak chha:
1.  **Measure (Naap):** Raw data.
    *   *Ex:* "Yo code ma 5 wota Error chha." (Yo number matra ho).
2.  **Metric (Tulana):** Comparison garna milne data.
    *   *Ex:* "Harek 1000 line code ma 2 wota error chha." (Rate nikalyo).
3.  **Indicator (Sanket):** Metric herera future predict garne.
    *   *Ex:* "Last month vanda error rate badhyo, so quality ghatdai chha."

**Direct vs Indirect Measure:**
*   **Direct:** LOC (Lines of Code), Execution Speed, Memory size.
*   **Indirect:** Functionality, Quality, Complexity, Reliability (Yiniharu sidhai napna mildaina).

---

### **1.7 Project Estimation (Numerical aauna sakcha)**

Software banauna kati paisa ra time lagcha? Estimate garne tarika:

1.  **LOC (Lines of Code):** Kati line code huncha andaj garne. (Purano tarika).
2.  **FP (Function Point):** Software le k k kaam (Input, Output, Query, Files) garcha tyo ganera estimate garne. (Language independent huncha).
3.  **COCOMO (Constructive Cost Model):** Boehm le banako formula.
    *   **Basic COCOMO:** Sano project ko lagi. Formula: $Effort = a(KLOC)^b$.
    *   **Intermediate/Detailed:** Dherai factor (Cost Drivers) herera calculate garne.

---

### **1.8 Software Risks (Jokhim)**

**Risk Management Process:**
1.  **Identification:** K galti huna sakcha? (Technology risk, People risk, Requirement risk).
2.  **Analysis:** Tyo risk aayo vane kati ghata lagcha? (Probability vs Impact).
3.  **Mitigation (Rokne):** Risk auna nadina k garne? (Ex: Staff chodne dar chha vane backup staff rakhne).
4.  **Monitoring:** Risk aayo ki ayena check garirakhne.

**RMMM Plan:**
*   **R**isk **M**itigation (Rokne).
*   **M**onitoring (Herne).
*   **M**anagement (Aaisakyo vane solve garne).

---

### **1.9 Software Engineering Ethics**

Engineer vayera k garna huncha, k hudaina? (ACM/IEEE Code of Ethics):
1.  **Public Interest:** Software le janata lai haani garnu vayena (Virus banauna vayena).
2.  **Client/Employer:** Boss ra Client ko gopya kura (Data) leak garnu vayena.
3.  **Product:** Quality ma compromise garnu vayena.
4.  **Judgment:** Ghus khayera naramro software pass garnu vayena.
5.  **Colleagues:** Sathi haru lai help garne, credit nakhosne.
